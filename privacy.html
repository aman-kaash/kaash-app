8. Security of Your Data

We implement the following reasonable security practices as required under Section 43A of the IT Act and the SPDI Rules, 2011:

  • All data in transit is encrypted using TLS 1.2 or higher
  • Firebase Firestore data is encrypted at rest by Google
  • Payment data is never stored on our servers — handled entirely by Razorpay's PCI-DSS-certified infrastructure
  • Payment verification uses HMAC-SHA256 signature validation (no client-side trust)
  • Premium status is set exclusively server-side via Firebase Admin SDK — cannot be self-granted by users
  • Firestore security rules enforce per-user data isolation — no user can read another user's data
  • Admin panel access is restricted to a single authenticated email address
  • Content Security Policy headers restrict script sources

In the event of a data breach that is likely to result in harm to you, we will notify affected users and the relevant authority within the time period prescribed under applicable law.
